How to run sudo apt-get update through proxy in Shell

Oct 20, 2015 · 1 minute read · Comments
linux I was struggling with this because I was configuring my http_proxy, https_proxy and ftp_proxy using export but when I was doing sudo apt-get udpate, nothing happened. Did some research on StackOverflow and found the solution. So I’m keeping the solution here for future reference. This problem happens because in some releases sudo is configured in such a way that all environment variables all cleared when running the command. My proxy didn’t have authentication so I did: export http_proxy="http://proxy.servername:port/" export https_proxy="http://proxy.servername:port/" export ftp_proxy="http://proxy.servername:port/" After that I did: visudo There is a line that states: Defaults env_reset After that line just add: Defaults env_keep = "http_proxy https_proxy ftp_proxy" Voilá.
